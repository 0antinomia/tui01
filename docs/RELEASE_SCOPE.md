# Release Scope

## 0.1.0 范围

这个版本的目标不是“功能很多”，而是“接入路径闭环”。

### 包含

- 配置驱动页面定义
- 宿主动作注册
- 参数化动作
- 配置与宿主校验
- shell 执行策略
- cwd/env 白名单
- 宿主日志与事件桥
- 最小宿主模板

### 明确不包含

- 复杂数据表格
- 树视图
- 多窗口布局系统
- 公开插件系统
- 稳定主题系统
- 完整国际化

## 0.1.0 的成功标准

- 新项目可以照模板接入
- 配置加载后能在启动前完成宿主校验
- 宿主可限制执行边界
- 动作执行可被宿主日志系统观测

## 下一阶段

进入下一阶段前，应优先做：

1. 至少一份真实项目接入样本
2. 一轮 API 收敛
3. 对 `AppConfig` 和 `RuntimeHost` 的兼容性重新审视
