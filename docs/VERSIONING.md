# 版本策略

## 当前状态

当前 crate 版本是 `0.3.0`。这表示：

- 项目已经具备一条相对清晰的宿主接入主路径
- 公开入口已经收敛到更小的范围
- API 仍处于持续调整阶段
- 现阶段不承诺稳定 API，也不承诺强兼容

## 现阶段如何看待版本号

在 `0.x` 阶段，版本号的意义更偏向“阶段性整理结果”，而不是稳定承诺：

- minor 版本仍然可能包含破坏性调整
- patch 版本应尽量只做小范围修正
- 版本变化不应被理解为已经进入长期稳定阶段

## 当前优先保证的方向

目前版本策略更关注这些事情：

1. 宿主接入路径清晰
2. 配置层和宿主层边界清楚
3. 动作执行链可预测
4. 自定义控件、主题和布局扩展点还能继续演进

这里说的主接入路径，指的是：

- `tui01::prelude`
- `tui01::field`
- `RuntimeHost`

## 暂时不保证的内容

目前不优先保证：

- 大范围 UI 组件 API 长期稳定
- 所有内部模块路径长期不变
- `spec / runtime / controls / components / host / app / infra` 的内部细节长期不变
- 历史兼容别名和旧入口会一直保留

## 使用建议

如果你准备把 `tui01` 用在真实项目里，建议：

- 固定你依赖的 crate 版本
- 优先参考 README、`docs/GETTING_STARTED.md` 和官方示例
- 避免依赖未在文档中明确承诺的内部模块细节
- 优先使用 `tui01::prelude`、`tui01::field` 和 `RuntimeHost`
